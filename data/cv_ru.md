## Ключевой опыт и реализованные проекты

### 1. Архитектура корпоративного DWH и аналитическая платформа («Quotas_analytic»)
* **Роль:** Lead Data Engineer & IT Architect.
* **Задача:** Провести рефакторинг и мигрировать распределенную экосистему Python ETL-скриптов, сохранявшую данные в CSV-файлы и базы данных Notion, из-за критического падения производительности Notion при росте объема данных. Реализовать сложную логику перекрестных связей для расчета фактического вылова водных биоресурсов (квот).
* **Решение:** Проектирование и развертывание централизованного хранилища данных (DWH) на базе PostgreSQL (SQLAlchemy 2.0 + миграции Alembic). Разработана сложная бизнес-логика «Трансферов квот», динамически вычисляющая фактического судовладельца и держателя квоты с учетом дат тайм-чартера судов. Интегрировано UN Comtrade Data API для импорта мировых цен по кодам HS, автоматизирован сбор данных подходов судов в порты через парсинг АМП. Настроено автоматическое обновление Notion-дашбордов в режиме Read-Only через Notion API. Оркестрация пайплайнов реализована в n8n на выделенном VPS.

### 2. Мультимодальный ИИ-ассистент и автоматизация для отелей («Nebo AI Assistant»)
* **Роль:** Backend Developer & IT Architect.
* **Задача:** Автоматизировать разбор технических инцидентов, заявок на ремонт и часто задаваемых вопросов (FAQ) от гостей и персонала в сети апарт-отелей «5 Небо» и «47 Небо». Разгрузить администраторов отеля и интегрировать ассистента с PMS.
* **Решение:** Разработано асинхронное FastAPI-приложение, интегрированное с Bnovo PMS API. Реализована гибридная RAG-система на базе ChromaDB и GigaChat с разделением промптов подStaff и Guest режимы. Внедрен модуль Computer Vision (GPT-4o / GigaChat Vision) для автоматической детекции дефектов оборудования в номерах по фотографиям с формированием CRM-тикетов. Интегрирован сервис Yandex SpeechKit для асинхронного STT-распознавания нативных голосовых сообщений Telegram (формат OGG OPUS) без перегрузки CPU сервера.
* **Безопасность и DevOps:** Контейнеризация приложения через Docker Compose. Внедрен механизм верификации вебхуков Telegram через кастомные заголовки `X-Telegram-Bot-Api-Secret-Token`. Реализован Feature Flag / Kill Switch (`MAINTENANCE_MODE`) для мгновенного перевода ассистента в режим обслуживания. Защита базы данных настроена через Row-Level Security (RLS).

### 3. Международный мультимодальный AI-анализатор конкурентов
* **Роль:** Full-Stack AI Engineer.
* **Задача:** Сбор и структурирование данных международных системных интеграторов судового оборудования связи (VSAT, LEO: Starlink Maritime, OneWeb) для извлечения информации об их сервисных хабах в портах, SLA и партнерских статусах (Elcome, Tototheo, AST).
* **Решение:** Создано десктопное приложение для macOS на базе PyQt6. Интегрирован runtime-парсер на Selenium в headless-режиме с подменой User-Agent и отключением автоматизации для беспрепятственного скрейпинга интерактивных карт локаций и списков партнеров конкурентов. Данные (включая встроенные PDF-брошюры и скриншоты) отправляются в мультимодальную модель GPT-4o с параметром `temperature=0` и принудительным `response_format={"type": "json_object"}` для исключения галлюцинаций. Приложение выдает строгий JSON-снимок с оценкой рисков, перечнем вендоров и списком портов присутствия инженеров конкурента. Сборка приложения выполнена через PyInstaller в нативный бандл `.app`.

### 4. R&D Кейс: Тонкое дообучение стиля общения локальных моделей (Fine-Tuning)
* **Роль:** AI Research Engineer.
* **Задача:** Отработка сценария полной изоляции конфиденциальных переписок VIP-гостей и кастомизации тональности речи (Tone of Voice) под стандарты пятизвездочного консьерж-сервиса.
* **Решение:** В изолированной среде Google Colab на видеокарте T4 GPU реализован пайплайн Fine-Tuning'а. Использована базовая русскоязычная модель Сбера (`rugpt3large_based_on_gpt2`) с применением 4-битного квантования через `BitsAndBytesConfig` для экономии памяти VRAM. Сконфигурирован LoRA-адаптер (`LoraConfig`, r=8, alpha=16) с обучением слоев внимания (`c_attn`). Обучение проведено с помощью библиотеки Hugging Face `trl` (`SFTTrainer` + `SFTConfig`) на кастомном синтаксическом датасете отеля. Обученный LoRA-адаптер успешно сохранен и подключен через `PeftModel` для локального инференса. По результатам проведен post-mortem анализ эффектов недообучения (underfitting) базовых моделей.
